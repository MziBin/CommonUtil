# LanguageManagerHelper 高优先级修复计划

## 修复目标

修复两个高优先级问题：
1. 内存泄漏（控件字典无限增长）
2. 添加线程安全支持

## 修复方案

### 1. 线程安全支持

**修改内容**：使用`Lazy<T>`实现线程安全的单例模式

```csharp
// 私有静态只读实例，使用Lazy<T>确保线程安全
private static readonly Lazy<LanguageManagerHelper> lazyInstance = 
    new Lazy<LanguageManagerHelper>(() => new LanguageManagerHelper());

// 公共静态属性，用于获取唯一实例
public static LanguageManagerHelper Instance => lazyInstance.Value;
```

**优点**：
- .NET Framework 4.0+ 内置支持
- 自动实现线程安全的延迟初始化
- 性能优于锁机制

### 2. 内存泄漏修复

**修改内容**：
1. 添加窗体关闭事件监听
2. 在窗体关闭时自动清理该窗体控件的引用
3. 实现IDisposable接口

```csharp
// 添加窗体关闭事件监听
private void SubscribeFormClosedEvent(Form form)
{
    form.FormClosed += OnFormClosed;
}

// 窗体关闭事件处理
private void OnFormClosed(object sender, FormClosedEventArgs e)
{
    if (sender is Form form)
    {
        RemoveFormControls(form);
    }
}

// 移除窗体及其控件的引用
private void RemoveFormControls(Form form)
{
    RemoveControlRecursive(form);
}

// 递归移除控件引用
private void RemoveControlRecursive(Control control)
{
    _controlOriginalTexts.Remove(control);
    
    foreach (Control childControl in control.Controls)
    {
        RemoveControlRecursive(childControl);
    }
}
```

**修改位置**：
- 在`InitializeFormControlOriginalTexts`方法中添加窗体关闭事件监听
- 实现`IDisposable`接口

### 3. 代码位置调整

1. 在类中添加`SubscribeFormClosedEvent`方法
2. 在`InitializeFormControlOriginalTexts`方法中调用`SubscribeFormClosedEvent`
3. 在类末尾添加`IDisposable`接口实现
