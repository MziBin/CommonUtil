## 8.栈

```csharp
// 1. 创建一个int类型的栈
        Stack<int> stack = new Stack<int>();

        // 2. 入栈（Push）：元素依次添加到栈顶
        stack.Push(10);  // 栈：[10]（栈顶是10）
        stack.Push(20);  // 栈：[10, 20]（栈顶是20）
        stack.Push(30);  // 栈：[10, 20, 30]（栈顶是30）
        Console.WriteLine($"栈中元素数量：{stack.Count}"); // 输出：3

        // 3. 查看栈顶（Peek）：不移除元素
        int top = stack.Peek();
        Console.WriteLine($"栈顶元素：{top}"); // 输出：3（栈仍为[10,20,30]）

        // 4. 出栈（Pop）：移除并返回栈顶元素
        int popped = stack.Pop();
        Console.WriteLine($"出栈元素：{popped}"); // 输出：3
        Console.WriteLine($"出栈后栈顶：{stack.Peek()}"); // 输出：2（栈变为[10,20]）

        // 5. 判断是否包含元素
        bool has10 = stack.Contains(10);
        Console.WriteLine($"栈中是否包含10：{has10}"); // 输出：True

        // 6. 清空栈
        stack.Clear();
        Console.WriteLine($"清空后元素数量：{stack.Count}"); // 输出：0
```

## 10.Task使用令牌取消和开始

```csharp
// 不要用静态cts，改为局部变量或实例变量，每次启动时新建
private CancellationTokenSource cts;
private bool isRunning = false; // 用更清晰的变量名表示“是否正在运行”
private Task currentTask; // 保存当前运行的Task，便于等待或处理
private void btn03_Click(object sender, EventArgs e)
{
    if (!isRunning)
    {
        // 1. 每次启动时创建新的取消令牌源
        cts = new CancellationTokenSource();
        CancellationToken token = cts.Token;

        // 2. 启动Task，保存到currentTask
        currentTask = Task.Run(async () =>
        {
            try
            {
                while (true)
                {
                    // 【关键】先检查取消，优先响应取消请求
                    token.ThrowIfCancellationRequested();

                    // 更新UI（确保在UI线程）
                    if (txt03.InvokeRequired)
                    {
                        txt03.Invoke(new Action(() =>
                        {
                            // 简化逻辑：直接在Invoke中处理数值累加
                            if (int.TryParse(txt03.Text, out int num))
                                txt03.Text = (num + 1).ToString();
                            else
                                txt03.Text = "1";
                        }));
                    }

                    // 【关键】Delay传入token，允许中途被取消
                    await Task.Delay(1000, token);
                }
            }
            catch (OperationCanceledException)
            {
                // 捕获取消异常，正常退出，不做处理（或记录日志）
                Console.WriteLine("Task已被取消");
            }
        }, token);

        isRunning = true;
        btn03.Text = "取消"; // 按钮文本提示当前状态
    }
    else
    {
        // 3. 取消Task
        if (cts != null)
        {
            cts.Cancel(); // 发送取消请求
            cts.Dispose(); // 释放资源
            cts = null;
        }
        isRunning = false;
        btn03.Text = "启动";
    }
}
```

## 17.IO字节流和文件流操作

在 C# 中，字节流（Byte Stream） 是基于System.IO.Stream抽象类的数据流，以字节为单位处理数据，是所有 IO 操作的基础。它适用于所有类型的文件（文本、图片、音频、视频等），尤其适合处理二进制数据。以下是字节流的核心使用方法、场景及示例：
一、字节流的核心类
字节流的操作依赖于Stream抽象类及其子类，最常用的有：
FileStream：操作磁盘文件的字节流（最常用）。
MemoryStream：操作内存数据的字节流（临时数据处理）。
NetworkStream：网络通讯的字节流（如 Socket 数据传输）



### FileStream 字节流

创建`FileStream`时需要指定文件路径、操作模式、访问权限等参数，常用构造函数：
    FileStream(string path, FileMode mode, FileAccess access, FileShare share)

* **关键参数说明**：
  * `path`：文件路径（如`"C:\\data\\file.bin"`）。
  * `FileMode`：文件打开 / 创建模式（决定如何处理文件，如新建、覆盖、追加等）：
    * `Create`：创建新文件，若已存在则覆盖。
    * `CreateNew`：创建新文件，若已存在则抛异常。
    * `Open`：打开现有文件，若不存在则抛异常。
    * `OpenOrCreate`：打开现有文件，若不存在则创建。
    * `Append`：打开文件并定位到末尾（仅用于写入，若不存在则创建）。
  * `FileAccess`：访问权限（读 / 写 / 读写）：
    * `Read`：仅读取。
    * `Write`：仅写入。
    * `ReadWrite`：读写。
  * `FileShare`：文件共享权限（其他进程可对文件执行的操作）：
    * `None`：不允许其他进程访问。
    * `Read`：允许其他进程读取。
      
      

//可读可写

`FileStream`支持随机访问（可跳转到文件任意位置读写），通过`Seek`方法设置当前位置：

```csharp
string filePath = "C:\\data\\test.bin";

using (FileStream fs = new FileStream(filePath, FileMode.OpenOrCreate, FileAccess.ReadWrite))
{
    // 1. 定位到文件第3个字节（从0开始）
    fs.Seek(3, SeekOrigin.Begin);

    // 2. 读取当前位置后的所有数据（暂存到内存）
    byte[] tempBuffer = new byte[fs.Length - fs.Position];
    fs.Read(tempBuffer, 0, tempBuffer.Length);

    // 3. 回到第3个字节位置，写入新数据
    fs.Seek(3, SeekOrigin.Begin);
    byte[] newData = new byte[] { 0x41, 0x42 }; // "AB"的ASCII码
    fs.Write(newData, 0, newData.Length);

    // 4. 写入之前暂存的数据（完成插入）
    fs.Write(tempBuffer, 0, tempBuffer.Length);
}
```



### 文件流

与字节流的区别：StreamWriter和StreamReader是 “文本流”，以字符为单位，处理编码；FileStream是 “字节流”，以字节为单位，适合二进制文件。写入文本时优先用StreamWriter，更简洁。



```csharp
string filePath = "demo.txt";
string content = "Hello, StreamReader & StreamWriter!";

// 1. 用StreamWriter写入文本
using (StreamWriter sw = new StreamWriter(filePath, false, Encoding.UTF8))
{
    sw.WriteLine(content); // 写入一行
}

// 2. 用StreamReader读取文本
using (StreamReader sr = new StreamReader(filePath, Encoding.UTF8))
{
    string readContent = sr.ReadLine(); // 读取一行
    Console.WriteLine("读取到的内容：" + readContent); // 输出：Hello, StreamReader & StreamWriter!
}
```



两者的构造函数都需要指定目标（文件路径或基础字节流）和编码，但`StreamWriter`多了 “追加模式” 的控制参数（因写入需区分 “覆盖” 或 “追加”）：

* `StreamReader`构造函数（无追加参数，读操作无需此概念）：
  
  ```csharp
  // 基于文件路径+编码
  new StreamReader(string path, Encoding encoding);
  // 基于基础字节流+编码
  new StreamReader(Stream stream, Encoding encoding);
  ```

* `StreamWriter`构造函数（有`append`参数，控制是否追加内容）：
  
  ```csharp
  // 基于文件路径+追加模式+编码（核心差异：append参数）
  new StreamWriter(string path, bool append, Encoding encoding);
  // 基于基础字节流+编码
  new StreamWriter(Stream stream, Encoding encoding);
  ```
  
  

#### 核心方法对比

| 功能分类      | `StreamReader`（读文本）常用方法          | `StreamWriter`（写文本）常用方法     |
| --------- | -------------------------------- | --------------------------- |
| **整体读写**  | `ReadToEnd()`：读取所有剩余文本           | 无对应方法（写入需主动调用`Write`系列）     |
| **逐行操作**  | `ReadLine()`：读取一行文本（到换行符）        | `WriteLine(string)`：写入一行并换行 |
| **单个字符**  | `Read()`：读取单个字符（返回编码值）           | `Write(char)`：写入单个字符        |
| **字符串操作** | 无（读字符串需通过`ReadLine`或`ReadToEnd`） | `Write(string)`：写入字符串（不换行）  |
| **资源控制**  | `Close()`/`Dispose()`：释放读资源      | `Close()`/`Dispose()`：释放写资源 |
| **特殊操作**  | `Peek()`：查看下一个字符（不移动位置）          | `Flush()`：强制刷新缓冲区到目标存储      |

  ```csharp
  // 示例：分块读取大文件
  using (StreamReader reader = new StreamReader("largefile.txt"))
  {
      string line;
      while ((line = reader.ReadLine()) != null) // 逐行读取
      {
          // 处理每行内容
      }
  }
  ```

## 20.序列化

序列化就是将对象的字段，属性值变成可存储文件的过程，而 [Serializable] 标签的作用是表示可以将这个对象 序列化的标记，而 JSON，XML等这些都已经不需要 [Serializable] 标签了。



在编程中，序列化（Serialization） 是指将对象的状态（即对象的字段、属性值） 转换为一种可存储或可传输的格式（如字节流、JSON 字符串、XML 文档等）的过程。其逆过程（将这种格式还原为原始对象）称为反序列化（Deserialization）。
核心目的
序列化的核心是解决 “对象状态的持久化” 和 “跨场景传输” 问题：
持久化：将对象状态保存到存储介质（如文件、数据库），以便后续程序重启后能恢复对象（例如游戏存档、配置保存）。
传输：在网络、进程、设备之间传输对象（例如 API 接口中客户端与服务器交换数据、分布式系统中不同节点传递信息）。
为什么需要序列化？
对象是程序在内存中的抽象概念（如User类的实例user，包含Name、Age等属性），但内存中的对象无法直接被存储到文件或通过网络发送 —— 必须将其转换为通用格式（如字节、文本）才能跨场景传递。
例如：
你不能直接把内存中的user对象 “扔” 到文件里，但可以将其序列化为 JSON 字符串（如{"Name":"张三","Age":20}），再保存到文件。
客户端不能直接把Order对象 “发送” 给服务器，但可以将其序列化为字节流，通过网络传输后，服务器再反序列化为Order对象处理。
常见序列化格式
不同场景下会选择不同的序列化格式，各有特点：


| 格式                   | 特点                                | 典型场景                |
| -------------------- | --------------------------------- | ------------------- |
| **JSON**             | 轻量级、易读、跨语言支持好（几乎所有编程语言都能解析）       | API 接口数据传输、配置文件     |
| **XML**              | 结构化强、支持复杂类型和命名空间，但较冗余             | 传统 WebService、配置文件  |
| **二进制**              | 紧凑（体积小）、效率高，但可读性差、通常不跨语言          | 本地文件存储（如游戏存档）、进程间通信 |
| **Protocol Buffers** | 谷歌开发的二进制格式，比 JSON 更紧凑、解析更快，跨语言支持好 | 高性能服务间通信            |
