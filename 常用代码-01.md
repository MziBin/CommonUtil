## 8.栈

```csharp
// 1. 创建一个int类型的栈
        Stack<int> stack = new Stack<int>();

        // 2. 入栈（Push）：元素依次添加到栈顶
        stack.Push(10);  // 栈：[10]（栈顶是10）
        stack.Push(20);  // 栈：[10, 20]（栈顶是20）
        stack.Push(30);  // 栈：[10, 20, 30]（栈顶是30）
        Console.WriteLine($"栈中元素数量：{stack.Count}"); // 输出：3

        // 3. 查看栈顶（Peek）：不移除元素
        int top = stack.Peek();
        Console.WriteLine($"栈顶元素：{top}"); // 输出：3（栈仍为[10,20,30]）

        // 4. 出栈（Pop）：移除并返回栈顶元素
        int popped = stack.Pop();
        Console.WriteLine($"出栈元素：{popped}"); // 输出：3
        Console.WriteLine($"出栈后栈顶：{stack.Peek()}"); // 输出：2（栈变为[10,20]）

        // 5. 判断是否包含元素
        bool has10 = stack.Contains(10);
        Console.WriteLine($"栈中是否包含10：{has10}"); // 输出：True

        // 6. 清空栈
        stack.Clear();
        Console.WriteLine($"清空后元素数量：{stack.Count}"); // 输出：0
```

## 10.Task使用令牌取消和开始

```csharp
// 不要用静态cts，改为局部变量或实例变量，每次启动时新建
private CancellationTokenSource cts;
private bool isRunning = false; // 用更清晰的变量名表示“是否正在运行”
private Task currentTask; // 保存当前运行的Task，便于等待或处理
private void btn03_Click(object sender, EventArgs e)
{
    if (!isRunning)
    {
        // 1. 每次启动时创建新的取消令牌源
        cts = new CancellationTokenSource();
        CancellationToken token = cts.Token;

        // 2. 启动Task，保存到currentTask
        currentTask = Task.Run(async () =>
        {
            try
            {
                while (true)
                {
                    // 【关键】先检查取消，优先响应取消请求
                    token.ThrowIfCancellationRequested();

                    // 更新UI（确保在UI线程）
                    if (txt03.InvokeRequired)
                    {
                        txt03.Invoke(new Action(() =>
                        {
                            // 简化逻辑：直接在Invoke中处理数值累加
                            if (int.TryParse(txt03.Text, out int num))
                                txt03.Text = (num + 1).ToString();
                            else
                                txt03.Text = "1";
                        }));
                    }

                    // 【关键】Delay传入token，允许中途被取消
                    await Task.Delay(1000, token);
                }
            }
            catch (OperationCanceledException)
            {
                // 捕获取消异常，正常退出，不做处理（或记录日志）
                Console.WriteLine("Task已被取消");
            }
        }, token);

        isRunning = true;
        btn03.Text = "取消"; // 按钮文本提示当前状态
    }
    else
    {
        // 3. 取消Task
        if (cts != null)
        {
            cts.Cancel(); // 发送取消请求
            cts.Dispose(); // 释放资源
            cts = null;
        }
        isRunning = false;
        btn03.Text = "启动";
    }
}
```

## 17.IO字节流和文件流操作

在 C# 中，字节流（Byte Stream） 是基于System.IO.Stream抽象类的数据流，以字节为单位处理数据，是所有 IO 操作的基础。它适用于所有类型的文件（文本、图片、音频、视频等），尤其适合处理二进制数据。以下是字节流的核心使用方法、场景及示例：
一、字节流的核心类
字节流的操作依赖于Stream抽象类及其子类，最常用的有：
FileStream：操作磁盘文件的字节流（最常用）。
MemoryStream：操作内存数据的字节流（临时数据处理）。
NetworkStream：网络通讯的字节流（如 Socket 数据传输）

### FileStream 字节流

创建 `FileStream`时需要指定文件路径、操作模式、访问权限等参数，常用构造函数：
    FileStream(string path, FileMode mode, FileAccess access, FileShare share)

* **关键参数说明**：
  * `path`：文件路径（如 `"C:\\data\\file.bin"`）。
  * `FileMode`：文件打开 / 创建模式（决定如何处理文件，如新建、覆盖、追加等）：
    * `Create`：创建新文件，若已存在则覆盖。
    * `CreateNew`：创建新文件，若已存在则抛异常。
    * `Open`：打开现有文件，若不存在则抛异常。
    * `OpenOrCreate`：打开现有文件，若不存在则创建。
    * `Append`：打开文件并定位到末尾（仅用于写入，若不存在则创建）。
  * `FileAccess`：访问权限（读 / 写 / 读写）：
    * `Read`：仅读取。
    * `Write`：仅写入。
    * `ReadWrite`：读写。
  * `FileShare`：文件共享权限（其他进程可对文件执行的操作）：
    * `None`：不允许其他进程访问。
    * `Read`：允许其他进程读取。

//可读可写

`FileStream`支持随机访问（可跳转到文件任意位置读写），通过 `Seek`方法设置当前位置：

```csharp
string filePath = "C:\\data\\test.bin";

using (FileStream fs = new FileStream(filePath, FileMode.OpenOrCreate, FileAccess.ReadWrite))
{
    // 1. 定位到文件第3个字节（从0开始）
    fs.Seek(3, SeekOrigin.Begin);

    // 2. 读取当前位置后的所有数据（暂存到内存）
    byte[] tempBuffer = new byte[fs.Length - fs.Position];
    fs.Read(tempBuffer, 0, tempBuffer.Length);

    // 3. 回到第3个字节位置，写入新数据
    fs.Seek(3, SeekOrigin.Begin);
    byte[] newData = new byte[] { 0x41, 0x42 }; // "AB"的ASCII码
    fs.Write(newData, 0, newData.Length);

    // 4. 写入之前暂存的数据（完成插入）
    fs.Write(tempBuffer, 0, tempBuffer.Length);
}
```

### 文件流

与字节流的区别：StreamWriter和StreamReader是 “文本流”，以字符为单位，处理编码；FileStream是 “字节流”，以字节为单位，适合二进制文件。写入文本时优先用StreamWriter，更简洁。

```csharp
string filePath = "demo.txt";
string content = "Hello, StreamReader & StreamWriter!";

// 1. 用StreamWriter写入文本
using (StreamWriter sw = new StreamWriter(filePath, false, Encoding.UTF8))
{
    sw.WriteLine(content); // 写入一行
}

// 2. 用StreamReader读取文本
using (StreamReader sr = new StreamReader(filePath, Encoding.UTF8))
{
    string readContent = sr.ReadLine(); // 读取一行
    Console.WriteLine("读取到的内容：" + readContent); // 输出：Hello, StreamReader & StreamWriter!
}
```

两者的构造函数都需要指定目标（文件路径或基础字节流）和编码，但 `StreamWriter`多了 “追加模式” 的控制参数（因写入需区分 “覆盖” 或 “追加”）：

* `StreamReader`构造函数（无追加参数，读操作无需此概念）：

  ```csharp
  // 基于文件路径+编码
  new StreamReader(string path, Encoding encoding);
  // 基于基础字节流+编码
  new StreamReader(Stream stream, Encoding encoding);
  ```
* `StreamWriter`构造函数（有 `append`参数，控制是否追加内容）：

  ```csharp
  // 基于文件路径+追加模式+编码（核心差异：append参数）
  new StreamWriter(string path, bool append, Encoding encoding);
  // 基于基础字节流+编码
  new StreamWriter(Stream stream, Encoding encoding);
  ```

#### 核心方法对比

| 功能分类             | `StreamReader`（读文本）常用方法                | `StreamWriter`（写文本）常用方法         |
| -------------------- | ------------------------------------------------- | ------------------------------------------ |
| **整体读写**   | `ReadToEnd()`：读取所有剩余文本                 | 无对应方法（写入需主动调用 `Write`系列） |
| **逐行操作**   | `ReadLine()`：读取一行文本（到换行符）          | `WriteLine(string)`：写入一行并换行      |
| **单个字符**   | `Read()`：读取单个字符（返回编码值）            | `Write(char)`：写入单个字符              |
| **字符串操作** | 无（读字符串需通过 `ReadLine`或 `ReadToEnd`） | `Write(string)`：写入字符串（不换行）    |
| **资源控制**   | `Close()`/`Dispose()`：释放读资源             | `Close()`/`Dispose()`：释放写资源      |
| **特殊操作**   | `Peek()`：查看下一个字符（不移动位置）          | `Flush()`：强制刷新缓冲区到目标存储      |

## 20.序列化

序列化就是将对象的字段，属性值变成可存储文件的过程，而 [Serializable] 标签的作用是表示可以将这个对象 序列化的标记，而 JSON，XML等这些都已经不需要 [Serializable] 标签了。

在编程中，序列化（Serialization） 是指将对象的状态（即对象的字段、属性值） 转换为一种可存储或可传输的格式（如字节流、JSON 字符串、XML 文档等）的过程。其逆过程（将这种格式还原为原始对象）称为反序列化（Deserialization）。
核心目的
序列化的核心是解决 “对象状态的持久化” 和 “跨场景传输” 问题：
持久化：将对象状态保存到存储介质（如文件、数据库），以便后续程序重启后能恢复对象（例如游戏存档、配置保存）。
传输：在网络、进程、设备之间传输对象（例如 API 接口中客户端与服务器交换数据、分布式系统中不同节点传递信息）。
为什么需要序列化？
对象是程序在内存中的抽象概念（如User类的实例user，包含Name、Age等属性），但内存中的对象无法直接被存储到文件或通过网络发送 —— 必须将其转换为通用格式（如字节、文本）才能跨场景传递。
例如：
你不能直接把内存中的user对象 “扔” 到文件里，但可以将其序列化为 JSON 字符串（如{"Name":"张三","Age":20}），再保存到文件。
客户端不能直接把Order对象 “发送” 给服务器，但可以将其序列化为字节流，通过网络传输后，服务器再反序列化为Order对象处理。
常见序列化格式
不同场景下会选择不同的序列化格式，各有特点：

| 格式                       | 特点                                                         | 典型场景                               |
| -------------------------- | ------------------------------------------------------------ | -------------------------------------- |
| **JSON**             | 轻量级、易读、跨语言支持好（几乎所有编程语言都能解析）       | API 接口数据传输、配置文件             |
| **XML**              | 结构化强、支持复杂类型和命名空间，但较冗余                   | 传统 WebService、配置文件              |
| **二进制**           | 紧凑（体积小）、效率高，但可读性差、通常不跨语言             | 本地文件存储（如游戏存档）、进程间通信 |
| **Protocol Buffers** | 谷歌开发的二进制格式，比 JSON 更紧凑、解析更快，跨语言支持好 | 高性能服务间通信                       |

### 返回类型的默认值：

`return default` 用于返回 **当前返回类型的默认值** ，遵循 C# 内置的默认值规则，核心用于 “无法确定具体返回值时，返回类型的默认状态”。

#### 1. C# 默认值规则（核心）

| 类型类别       | 默认值                           | 示例                          |
| -------------- | -------------------------------- | ----------------------------- |
| 引用类型       | `null`                         | `string` → `null`        |
| 简单值类型     | 零值（数值 0、bool=false 等）    | `int` → 0，`bool`→false |
| 自定义结构体   | 所有字段均为各自类型的默认值     | `Point { X:0, Y:0 }`        |
| 可空值类型     | `null`（`HasValue = false`） | `int?` → `null`          |
| 泛型类型参数   | 对应类型的默认值                 | `T` → `default(T)`       |
| Task/ValueTask | 已完成的空任务（非 null）        | `Task<int>` → default      |

#### 2. 核心适用场景

##### 场景 1：泛型方法（最常用）

泛型方法中无法预知具体返回类型，用 `return default` 返回对应类型的默认值。

```csharp
// 泛型方法：查找列表元素，未找到返回默认值
public T Find<T>(List<T> list, Predicate<T> match)
{
    foreach (var item in list)
    {
        if (match(item)) return item;
    }
    return default; // 等价于 return default(T);
}

// 调用示例
var intList = new List<int> { 1, 2, 3 };
var notFoundInt = Find(intList, x => x == 4); // 返回 0（int默认值）

var strList = new List<string> { "a", "b" };
var notFoundStr = Find(strList, x => x == "c"); // 返回 null（string默认值）
```


## 23.goto和ManualResetEvent

goto：*goto*语句用于无条件地跳转到方法内的带标签的语句。这个标签位于可执行语句之前，并以冒号（:）结尾。*goto*语句可以向前或向后跳转，可以跳出循环或跳入循环。

在此示例中，使用*goto*语句创建了一个循环，计算从1到100的累加和。每次迭代时，如果变量*number*小于或等于100，则将其值累加到变量*sum*中，并跳转回标签*StartLoop*继续循环。当变量*number*大于100时，循环结束，并输出累加和。

 **使用建议** ：尽管可以使用*goto*语句进行快速跳转，但应谨慎使用以避免复杂化代码逻辑。考虑使用其他控制流结构（如循环和条件语句）来实现更清晰、更可维护的代码结构。

```csharp
int sum = 0;
int number = 1;
StartLoop:
if (number <= 100)
{
sum += number;
number++;
goto StartLoop;
}
Console.WriteLine($"The sum of numbers from 1 to 100 is {sum}.");

```

ManualResetEvent：可以理解为一个手动断点，通过不同时机来开启或者设置断点，不要再主线程设置，这样主线程会一直等待。可以用来蜂鸣器的开关。

**ManualResetEvent** 是 .NET 中的线程同步机制，用于在线程之间提供信号通知。它允许多个线程等待信号，并在信号发出后同时被唤醒。与 **AutoResetEvent** 不同，ManualResetEvent 在被设置为信号状态后会保持该状态，直到显式调用 *Reset()* 方法。

ManualResetEvent 的构造函数接受一个布尔值，表示初始状态：

* *true* ：初始为信号状态，线程不会阻塞。
* *false* ：初始为非信号状态，线程会阻塞。

3个使用方法

```csharp
mre.WaitOne(); // 等待信号

mre.Reset(); // 重置为非信号状态

mre.Set(); // 设置为信号状态，释放所有线程
```

以下是一个简单的代码示例，展示如何使用 ManualResetEvent 控制线程同步：

```csharp
using System;
using System.Threading;
class Program
{
   static ManualResetEvent mre = new ManualResetEvent(false);
   static void Main()
   {
       // 创建多个线程
       for (int i = 0; i < 3; i++)
       {
           Thread t = new Thread(ThreadProc);
           t.Name = $"Thread_{i}";
           t.Start();
       }
       Console.WriteLine("按下 Enter 键以释放所有线程...");
       Console.ReadLine();
       mre.Set(); // 设置为信号状态，释放所有线程
       Console.WriteLine("按下 Enter 键以重置事件...");
       Console.ReadLine();
       mre.Reset(); // 重置为非信号状态
       // 启动一个新线程，观察其阻塞行为
       Thread t4 = new Thread(ThreadProc);
       t4.Name = "Thread_4";
       t4.Start();
       Console.WriteLine("按下 Enter 键以结束程序...");
       Console.ReadLine();
       mre.Set(); // 再次设置为信号状态，释放阻塞线程
   }
   static void ThreadProc()
   {
       Console.WriteLine($"{Thread.CurrentThread.Name} 开始并等待信号...");
       mre.WaitOne(); // 等待信号
       Console.WriteLine($"{Thread.CurrentThread.Name} 收到信号并结束。");
   }
}
```

## 24.定时器

在 C# 中，不同场景下需选用不同类型的定时器，核心分为 **UI 专用定时器** 和 **后台/通用定时器** 两大类，以下是主流定时器的使用方法、适用场景及核心差异：

### 一、核心定时器类型总览

| 定时器类型                                   | 适用场景             | 触发线程              | 精度        | 核心特点                                                                |
| -------------------------------------------- | -------------------- | --------------------- | ----------- | ----------------------------------------------------------------------- |
| `System.Windows.Forms.Timer`               | WinForm 桌面应用     | UI 线程               | 低（~55ms） | 简单，无需线程安全处理 UI                                               |
| `System.Windows.Threading.DispatcherTimer` | WPF 桌面应用         | UI 线程（Dispatcher） | 低（~15ms） | 适配 WPF 消息循环                                                       |
| `System.Timers.Timer`                      | 通用后台任务（非UI） | 线程池线程            | 中（~10ms） | 封装完善，支持自动重启，可能**重入** （上一次没跑完下一次又触发） |
| `System.Threading.Timer`                   | 高性能/轻量后台任务  | 线程池线程            | 高（~1ms）  | 最轻量化，无多余封装，可能**重入** （上一次没跑完下一次又触发）   |

### 二、分类型使用示例

#### 1. System.Windows.Forms.Timer（WinForm 专用）

**适用场景**：WinForm 中更新 UI（如倒计时、进度条刷新），依赖 Windows 消息循环，仅在 UI 线程触发。

**使用步骤**：

```csharp
using System;
using System.Windows.Forms;

public partial class Form1 : Form
{
    private Timer _formTimer;
    private int _count = 0;

    public Form1()
    {
        InitializeComponent();
  
        // 1. 创建定时器
        _formTimer = new Timer();
        // 2. 设置定时间隔（毫秒）
        _formTimer.Interval = 1000; // 1秒触发一次
        // 3. 绑定触发事件
        _formTimer.Tick += FormTimer_Tick;
        // 4. 启动定时器
        _formTimer.Start();
    }

    // 定时器触发事件（UI 线程执行）
    private void FormTimer_Tick(object sender, EventArgs e)
    {
        _count++;
        // 直接更新UI，无需Invoke（因为在UI线程）
        lblCount.Text = $"计数：{_count}";
  
        // 示例：5秒后停止定时器
        if (_count >= 5)
        {
            _formTimer.Stop();
            _formTimer.Dispose(); // 释放资源，避免内存泄漏
        }
    }

    // 窗体关闭时释放定时器
    protected override void OnFormClosed(FormClosedEventArgs e)
    {
        _formTimer?.Dispose();
        base.OnFormClosed(e);
    }
}
```

**注意事项**：

- 仅能在 WinForm 项目中使用，控制台/ASP.NET 中无效；
- 间隔最小值约 55ms（受 Windows 消息循环限制）；
- 若事件处理耗时超过间隔，会阻塞 UI 线程（如卡死界面）。

#### 2. System.Windows.Threading.DispatcherTimer（WPF 专用）

**适用场景**：WPF 中更新 UI（如实时数据展示），通过 `Dispatcher` 绑定 UI 线程。

**使用步骤**：

```csharp
using System;
using System.Windows;
using System.Windows.Threading;

public partial class MainWindow : Window
{
    private DispatcherTimer _dispatcherTimer;
    private int _count = 0;

    public MainWindow()
    {
        InitializeComponent();
  
        // 1. 创建定时器
        _dispatcherTimer = new DispatcherTimer();
        // 2. 设置间隔（毫秒）
        _dispatcherTimer.Interval = TimeSpan.FromSeconds(1);
        // 3. 绑定事件
        _dispatcherTimer.Tick += DispatcherTimer_Tick;
        // 4. 启动
        _dispatcherTimer.Start();
    }

    private void DispatcherTimer_Tick(object sender, EventArgs e)
    {
        _count++;
        // 直接更新WPF UI控件
        txtCount.Text = $"计数：{_count}";
  
        if (_count >= 5)
        {
            _dispatcherTimer.Stop();
            _dispatcherTimer = null; // 释放
        }
    }
}
```

**注意事项**：

- 依赖 WPF 的 `Dispatcher`，需在 UI 线程创建；
- 若事件处理耗时过长，会导致 UI 卡顿。

#### 3. System.Timers.Timer（通用后台定时器）

**适用场景**：非 UI 后台任务（如定时备份、数据同步），线程池线程触发，需注意线程安全。

**使用步骤**：

```csharp
using System;
using System.Timers;
// 若需更新UI，需引入对应命名空间（如WinForm：System.Windows.Forms）

class BackgroundTimerDemo
{
    private static Timer _systemTimer;
    private static int _count = 0;

    static void Main()
    {
        // 1. 创建定时器
        _systemTimer = new Timer();
        // 2. 设置间隔（毫秒）
        _systemTimer.Interval = 1000;
        // 3. 启用自动重启（默认true，触发后自动重置）
        _systemTimer.AutoReset = true;
        // 4. 绑定触发事件
        _systemTimer.Elapsed += SystemTimer_Elapsed;
        // 5. 启动定时器
        _systemTimer.Start();

        Console.WriteLine("定时器已启动，按任意键停止...");
        Console.ReadKey();
  
        // 停止并释放
        _systemTimer.Stop();
        _systemTimer.Dispose();
    }

    // 触发事件（线程池线程执行）
    private static void SystemTimer_Elapsed(object sender, ElapsedEventArgs e)
    {
        _count++;
        Console.WriteLine($"后台计数：{_count} | 线程ID：{System.Threading.Thread.CurrentThread.ManagedThreadId}");

        // 【关键】若需更新UI，需通过Invoke切换到UI线程（以WinForm为例）
        // Form1.Instance.Invoke(new Action(() => {
        //     Form1.Instance.lblCount.Text = $"计数：{_count}";
        // }));
    }
}
```

**注意事项**：

- `Elapsed` 事件在**线程池线程**执行，直接操作 UI 会抛跨线程异常；
- `AutoReset = false` 时，定时器仅触发一次；
- 需手动调用 `Dispose()` 释放资源，避免内存泄漏。

#### 4. System.Threading.Timer（高性能后台定时器）

**适用场景**：高性能、轻量后台任务（如高频数据采集），无多余封装，灵活性最高。

##### **1.基础使用步骤**：

```csharp
using System;
using System.Threading;

class HighPerformanceTimerDemo
{
    private static Timer _threadTimer;
    private static int _count = 0;

    static void Main()
    {
        // 1. 创建定时器
        // 参数：
        // - 回调方法
        // - 传递给回调的参数（可null）
        // - 延迟启动时间（0=立即启动）
        // - 定时间隔（毫秒）
        _threadTimer = new Timer(ThreadTimer_Callback, null, 0, 1000);

        Console.WriteLine("高性能定时器已启动，按任意键停止...");
        Console.ReadKey();
  
        // 停止定时器（Dispose会自动停止）
        _threadTimer.Dispose();
    }

    // 回调方法（线程池线程执行）
    private static void ThreadTimer_Callback(object state)
    {
        _count++;
        Console.WriteLine($"高性能计数：{_count} | 线程ID：{Thread.CurrentThread.ManagedThreadId}");

        // 示例：5次后停止定时器（通过修改间隔为Timeout.Infinite）
        if (_count >= 5)
        {
            _threadTimer.Change(Timeout.Infinite, Timeout.Infinite);
        }
    }
}
```

**注意事项**：

- 最轻量化，无 `Start/Stop` 方法，通过 `Change` 调整定时参数；
- `Change(int dueTime, int period)`：
  - `dueTime`：延迟启动时间（`Timeout.Infinite`=不启动）；
  - `period`：触发间隔（`Timeout.Infinite`=仅触发一次）；
- 回调执行时间若超过间隔，会导致回调重叠（需自行加锁控制）。

##### 2.启动暂停重新开始

`System.Threading.Timer` **支持手动控制启动、停止（暂停）和重启**，但它没有直接的 `Start()`/`Stop()` 方法，而是通过 `Change()` 方法调整定时参数来实现这些操作——这是它和 `System.Timers.Timer` 最核心的区别（后者有显式的 `Start/Stop`）。

核心原理

`System.Threading.Timer` 的 `Change(dueTime, period)` 方法是控制逻辑的核心，两个参数的含义：

| 参数        | 作用                                                                                                                                            |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| `dueTime` | 「延迟多久后首次触发」（毫秒）：`<br>`- `0` = 立即触发；`<br>`- `Timeout.Infinite` = 永不触发（停止）；`<br>`- 正数 = 延迟N毫秒触发。 |
| `period`  | 「首次触发后，每次触发的间隔」（毫秒）：`<br>`- `>0` = 固定间隔触发；`<br>`- `Timeout.Infinite` = 仅触发一次。                          |

通过调整这两个参数，就能实现「启动、停止、重启、单次触发、循环触发」等所有操作。

手动控制的完整示例

下面演示如何手动实现 **停止（暂停）、启动、重启** 逻辑：

```csharp
using System;
using System.Threading;

class ThreadingTimerControlDemo
{
    // 定义定时器实例（类级别，避免被GC回收）
    private static Timer _threadTimer;
    private static int _count = 0;

    static void Main()
    {
        Console.WriteLine("=== System.Threading.Timer 手动控制示例 ===");
  
        // 1. 初始化定时器（初始状态：暂停，永不触发）
        // 参数：回调方法、自定义参数、初始延迟（Infinite=暂停）、触发间隔（Infinite）
        _threadTimer = new Timer(TimerCallback, null, Timeout.Infinite, Timeout.Infinite);

        // 手动控制逻辑
        Console.WriteLine("输入指令：start（启动）| stop（停止）| restart（重启）| exit（退出）");
        string input;
        while ((input = Console.ReadLine()?.Trim().ToLower()) != "exit")
        {
            switch (input)
            {
                case "start":
                    StartTimer();
                    break;
                case "stop":
                    StopTimer();
                    break;
                case "restart":
                    RestartTimer();
                    break;
                default:
                    Console.WriteLine("无效指令！");
                    break;
            }
        }

        // 退出前彻底释放定时器
        _threadTimer.Dispose();
        Console.WriteLine("定时器已释放，程序退出");
    }

    // 手动启动定时器（1秒后首次触发，后续每1秒触发一次）
    private static void StartTimer()
    {
        // Change(延迟启动时间, 循环间隔)
        _threadTimer.Change(1000, 1000); 
        Console.WriteLine("定时器已启动（1秒后首次触发，每秒循环）");
    }

    // 手动停止（暂停）定时器（永不触发，保留实例可重启）
    private static void StopTimer()
    {
        // Change(永不触发, 永不触发) → 停止所有后续触发
        _threadTimer.Change(Timeout.Infinite, Timeout.Infinite);
        Console.WriteLine($"定时器已停止 | 当前计数：{_count}");
    }

    // 手动重启定时器（立即触发，重置计数）
    private static void RestartTimer()
    {
        _count = 0; // 重置计数
        _threadTimer.Change(0, 1000); // 立即触发，后续每秒一次
        Console.WriteLine("定时器已重启（立即触发，每秒循环）");
    }

    // 定时器回调方法（线程池线程执行）
    private static void TimerCallback(object state)
    {
        _count++;
        Console.WriteLine($"[{DateTime.Now:HH:mm:ss}] 触发计数：{_count} | 线程ID：{Thread.CurrentThread.ManagedThreadId}");
    }
}
```

关键操作说明

| 操作         | 实现代码                                                      | 效果说明                                                       |
| ------------ | ------------------------------------------------------------- | -------------------------------------------------------------- |
| 初始暂停     | `new Timer(回调, null, Timeout.Infinite, Timeout.Infinite)` | 定时器创建后不触发，处于“停止”状态。                         |
| 启动         | `_threadTimer.Change(1000, 1000)`                           | 延迟1秒首次触发，之后每1秒触发一次（循环）。                   |
| 立即启动     | `_threadTimer.Change(0, 1000)`                              | 无延迟立即触发，之后每1秒触发一次。                            |
| 停止（暂停） | `_threadTimer.Change(Timeout.Infinite, Timeout.Infinite)`   | 停止所有后续触发，定时器实例保留，可再次调用 `Change` 重启。 |
| 单次触发     | `_threadTimer.Change(1000, Timeout.Infinite)`               | 延迟1秒触发一次，之后不再触发（相当于“一次性定时器”）。      |
| 彻底释放     | `_threadTimer.Dispose()`                                    | 销毁定时器实例，释放资源，**无法再重启**（需重新new）。  |

注意事项

1. **避免被GC回收**：若定时器实例是局部变量，容易被垃圾回收导致定时器失效——务必将其声明为**类级别字段**（如示例中 `_threadTimer`）。
2. **线程安全**：`Change()` 方法是线程安全的，可在任意线程调用（如UI线程、回调线程）；但回调方法仍在**线程池线程**执行，操作UI需通过 `Invoke` 切换线程。
3. **停止≠释放**：`Change(Timeout.Infinite, ...)` 只是暂停触发，定时器实例仍存在；只有调用 `Dispose()` 才会彻底释放资源（释放后无法重启）。
4. **回调重叠**：若回调方法执行耗时超过 `period` 间隔，会导致多个回调在不同线程池线程同时执行——需手动加锁（如 `lock`）避免并发问题。

### 三、关键注意事项

1. **线程安全**：
   - UI 定时器（Forms/Dispatcher）：无需线程安全处理，直接操作 UI；
   - 后台定时器（Timers/Threading）：操作 UI 需通过 `Control.Invoke`（WinForm）或 `Dispatcher.Invoke`（WPF）。
2. **资源释放**：
   - 所有定时器均需调用 `Dispose()` 释放资源（尤其是长生命周期程序）；
   - 桌面应用中，窗体关闭时务必释放定时器，避免内存泄漏。
3. **精度问题**：
   - 不要依赖定时器实现高精度计时（如毫秒级），需用 `Stopwatch` 辅助；
   - 后台定时器受系统负载影响，间隔可能有微小偏差。
4. **Timer定时器避免回调重叠**：
   - 若定时器事件处理耗时超过间隔，可设置 `System.Timers.Timer.SynchronizingObject` 或手动加锁（如 `lock`）,避免重复进入。

### 四、选型建议

- 做 WinForm/WPF UI 定时更新 → 选对应 UI 定时器；
- 简单后台任务 → 选 `System.Timers.Timer`（封装完善）；
- 高性能/高频后台任务 → 选 `System.Threading.Timer`（轻量化）；
- 控制台/ASP.NET 应用 → 优先 `System.Threading.Timer`（无 UI 依赖）。

## 25. 下划线 _

`_` 是 C# 中多用途的语法元素，核心分为**语法特性**和**编码规范**两类，以下是高频使用场景：

1. 丢弃变量（Discard）—— C# 7.0+ 核心特性

用于 **显式忽略不需要的返回值 / 变量** ，编译器不会为丢弃变量分配内存，仅表示 “该值无需使用”，是最常用的语法场景。

 **适用场景 & 示例** ：

```csharp
// 1. 元组解构：仅取第二个值，忽略第一个
var (_, age) = GetPersonInfo(); // _ 忽略姓名字段
Console.WriteLine($"年龄：{age}");

// 2. out参数忽略：仅判断解析成功，不关心解析结果
if (int.TryParse("123", out _)) 
{
    Console.WriteLine("解析成功");
}

// 3. 异步方法“火并忘”：忽略Task返回值（无需等待结果）
_ = DoAsyncWork(); 

// 4. 模式匹配：仅判断类型，忽略具体值
object obj = 123;
if (obj is int _) // 仅校验是否为int类型，忽略数值
{
    Console.WriteLine("是整数");
}

// 辅助方法
(string Name, int Age) GetPersonInfo() => ("张三", 25);
async Task DoAsyncWork() => await Task.Delay(1000);
```
